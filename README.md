# Watch Service

Кроссплатформенный файловый наблюдатель на Go. Отслеживает изменения файлов в указанных директориях и выполняет команды при событиях.

## Особенности

- **Кроссплатформенность**: работает на Windows, Linux, macOS
- **0 зависимостей**: только стандартная библиотека Go
- **Пакетная обработка**: выполнение команды 1 раз для всех изменённых файлов
- **Настраиваемый интервал**: задержка перед выполнением команды (debounce)
- **Рекурсивное отслеживание**: поддержка поддиректорий
- **Параллельное выполнение**: неограниченный пул команд

## Установка

```bash
go build -o watcher ./cmd/watcher
```

## Использование

```bash
./watcher -config config.toml
```

### Флаги

- `-config`: путь к файлу конфигурации (по умолчанию `config.toml`)

## Конфигурация

Файл конфигурации в формате TOML:

```toml
# Задержка перед выполнением команды (дебаунсинг)
delay = "500ms"

# Максимальное количество параллельных команд (0 = неограниченно)
max_parallel = 0

[[watch]]
path = "./watch_folder"
recursive = true
command = "cmd"
args = ["/C", "echo Обработано файлов: {WATCH_FILE_COUNT}"]
timeout = "60s"
```

### Опции

#### Глобальные

| Параметр | Тип | Описание |
|----------|-----|----------|
| `delay` | duration | Задержка перед выполнением команды |
| `max_parallel` | int | Максимум параллельных команд (0 = без ограничений) |

#### Watch

| Параметр | Тип | Описание |
|----------|-----|----------|
| `path` | string | Путь для отслеживания |
| `recursive` | bool | Рекурсивное отслеживание поддиректорий |
| `command` | string | Команда для выполнения |
| `args` | []string | Аргументы команды |
| `timeout` | duration | Таймаут выполнения команды |

### Переменные окружения

Команда получает переменные окружения:

| Переменная | Описание |
|------------|----------|
| `WATCH_FILES` | Список изменённых файлов (разделитель `;`) |
| `WATCH_FILE_COUNT` | Количество изменённых файлов |
| `WATCH_EVENT_TYPE` | Тип события (CREATE, WRITE, REMOVE, RENAME) |

### Плейсхолдеры в аргументах

| Плейсхолдер | Описание |
|-------------|----------|
| `{file}` | Путь к первому файлу (для совместимости) |
| `{files}` | Список всех файлов через пробел |
| `{WATCH_FILE_COUNT}` | Количество файлов |

## Примеры

### Сжатие новых файлов

```toml
[[watch]]
path = "./uploads"
recursive = true
command = "gzip"
args = ["-f", "{file}"]
timeout = "30s"
```

### Уведомление об изменениях

```toml
[[watch]]
path = "./src"
recursive = true
command = "notify-send"
args = ["Изменено файлов: {WATCH_FILE_COUNT}"]
timeout = "10s"
```

### Автоматическая сборка

```toml
[[watch]]
path = "./src"
recursive = true
command = "go"
args = ["build", "./..."]
timeout = "60s"
```

### Синхронизация файлов

```toml
[[watch]]
path = "./docs"
recursive = true
command = "rsync"
args = ["-av", "{files}", "server:/backup"]
timeout = "120s"
```

## Архитектура

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│   Watcher   │────▶│  Debouncer   │────▶│   Runner    │
│  (polling)  │     │   (batch)    │     │  (parallel) │
└─────────────┘     └──────────────┘     └─────────────┘
```

1. **Watcher**: опрашивает файловую систему каждые 500ms
2. **Debouncer**: агрегирует события в течение `delay`
3. **Runner**: выполняет команду один раз для всех файлов
